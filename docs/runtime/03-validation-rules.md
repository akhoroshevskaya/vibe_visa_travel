# Validation Rules

## Назначение

**Validation Rules** определяет минимальный обязательный набор правил, которым должна соответствовать архитектурная документация проекта **V!BE Business OS** перед переходом к реализации.

Документ устанавливает единые критерии проверки архитектуры и позволяет подтвердить готовность системы к следующему этапу жизненного цикла проекта.

---

## Цели проверки

Архитектурная проверка выполняется для подтверждения:

- полноты архитектурной документации;
- согласованности архитектурных решений;
- отсутствия логических противоречий;
- соответствия утвержденной **Architecture Baseline**;
- готовности архитектуры к реализации.

---

## Объекты проверки

Проверке подлежат:

- архитектурные документы;
- структура репозитория;
- архитектурные артефакты;
- связи между документами;
- обязательные метаданные;
- статусы документов;
- архитектурные зависимости.

---

## Обязательные проверки

Перед началом реализации должны быть выполнены следующие проверки.

### Документация

- Все обязательные архитектурные документы существуют.
- Все документы имеют уникальный идентификатор.
- Все документы содержат обязательные метаданные.
- Все документы имеют актуальный статус.

### Структура

- Структура репозитория соответствует Architecture Baseline.
- Все документы расположены в утвержденных каталогах.
- Отсутствуют дублирующие архитектурные документы.

### Зависимости

- Все ссылки указывают на существующие документы.
- Отсутствуют циклические зависимости.
- Все зависимости соответствуют утвержденной архитектуре.

### Архитектура

- Architecture Baseline имеет статус **Approved**.
- Критические архитектурные решения утверждены.
- Архитектура не содержит известных блокирующих противоречий.

---

## Критические ошибки

Следующие ошибки блокируют переход к реализации:

- отсутствует обязательный архитектурный документ;
- отсутствуют обязательные метаданные;
- нарушена структура репозитория;
- обнаружены циклические зависимости;
- Architecture Baseline не утверждена;
- нарушены архитектурные зависимости;
- обнаружены противоречащие друг другу архитектурные решения.

При наличии хотя бы одной критической ошибки результат проверки считается отрицательным.

---

## Некритичные замечания

Следующие замечания не блокируют реализацию, но должны быть устранены в процессе сопровождения архитектуры:

- устаревшие ссылки;
- неполные описания;
- отсутствие примеров;
- необходимость обновления README;
- необходимость обновления PROJECT_STATUS;
- рекомендации по улучшению структуры документации.

Все подобные замечания фиксируются в **Technical Debt** или **Architecture Backlog**.

---

## Результаты проверки

По итогам проверки присваивается один из следующих статусов.

| Статус | Значение |
|---------|----------|
| **PASS** | Архитектура соответствует требованиям и готова к реализации. |
| **PASS WITH WARNINGS** | Обнаружены некритичные замечания. Реализация может быть продолжена. |
| **FAILED** | Обнаружены критические ошибки. Переход к реализации запрещен до их устранения. |

---

## Использование результатов проверки

Результаты Validation используются для:

- принятия решения о начале реализации;
- проведения Architecture Review;
- обновления PROJECT_STATUS;
- контроля качества архитектуры;
- подготовки следующих этапов разработки.
