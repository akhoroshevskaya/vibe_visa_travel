# Назначение

Business OS Architecture Repository содержит архитектурную документацию операционного слоя компании.

Repository является продолжением Business Architecture и описывает цифровую операционную модель компании независимо от конкретной технологической платформы.

Business OS реализует утвержденную Business Architecture, но не изменяет ее.

# Правила проектирования

Business OS развивается по принципу Evolutionary Architecture.

Каждый архитектурный объект развивается независимо при сохранении совместимости с утвержденной Business Architecture.

Новые решения принимаются пакетами (Design Batch).

Изменения фиксируются через Architecture Review и Git Commit.
