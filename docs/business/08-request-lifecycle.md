# 1. Purpose

Request Lifecycle описывает путь обращения клиента от первого контакта до создания одного или нескольких Case.

Документ фиксирует:

- основные этапы жизненного цикла Request;
- бизнес-события;
- точки принятия решений;
- создаваемые бизнес-сущности;
- условия завершения жизненного цикла Request.

Request Lifecycle является связующим звеном между CRM, Finance и Operations.

---

# 2. Уровни зрелости Request

Жизненный цикл Request проходит три уровня зрелости.

## Level 1 — Interest

Клиент проявляет интерес к услугам компании.

На этом этапе происходит знакомство, квалификация и принятие решения о сотрудничестве.

Коммерческие отношения еще не начались.

---

## Level 2 — Commercial Agreement

Клиент принимает предложение компании.

Формируются первые коммерческие обязательства:

- подтверждение сотрудничества;
- принятие оферты;
- создание первой Financial Transaction.

---

## Level 3 — Execution

После получения первой коммерческой оплаты создается один или несколько Case.

Request завершает свою основную функцию и становится контекстом для дальнейшего сопровождения клиента.

Операционная работа переносится в Workflow каждого Case.

# 4. Основные принципы

## Request не является услугой

Request фиксирует обращение клиента.

Оказание услуги начинается только после создания Case.

---

## Один Request может породить несколько Case

Например:

- семья из нескольких человек;
- несколько самостоятельных услуг;
- повторные продукты в рамках одного обращения.

---

## Case является точкой начала операционной деятельности

После создания Case дальнейшая работа выполняется через Workflow.

Request больше не управляет выполнением услуги.

---

## Financial Transaction является границей между Interest и Execution

Первая коммерческая оплата переводит Request из стадии переговоров в стадию выполнения обязательств.

Именно после этого создается первый Case.

# 5. Lifecycle Phases

## Phase 1 — New Request

### Purpose

Зафиксировать первое обращение потенциального клиента.

---

### Business Events

- получено новое обращение;
- создан Request;
- назначен Manager;
- определен Primary Contact.

---

### Created Artifacts

- Request
- Client (если новый)
- Manager Assignment

---

### Result

Request зарегистрирован и готов к квалификации.

---

## Phase 2 — Qualification

### Purpose

Определить потребности клиента и подобрать оптимальное решение.

---

### Business Events

- проведена консультация;
- собрана необходимая информация;
- определены участники обращения;
- выбран подходящий Product.

---

### Created Artifacts

- список участников Request;
- рекомендации по Product;
- история Communication.

---

### Result

Request готов к формированию коммерческого предложения.

---

## Phase 3 — Waiting Decision

### Purpose

Ожидание решения клиента.

---

### Business Events

- отправлено коммерческое предложение;
- отправлена оферта;
- ожидается ответ клиента.

---

### Automation

Через 7 дней без ответа отправляется уважительное напоминание.

Через 30 дней Request автоматически закрывается со статусом **Closed – No Response**.

---

### Result

Request ожидает решения клиента.

---

## Phase 4 — Ready To Start

### Purpose

Подготовить начало коммерческих отношений.

---

### Business Events

- клиент подтвердил намерение работать;
- согласованы условия;
- подготовлены документы для начала сотрудничества.

---

### Result

Компания готова перейти к коммерческому этапу.

---

## Phase 5 — Commercial Agreement

### Purpose

Зафиксировать возникновение коммерческих обязательств.

---

### Business Events

- клиент принимает оферту;
- подтверждается согласие на обработку персональных данных;
- создается первая Financial Transaction.

---

### Created Artifacts

- принятое согласие с офертой;
- согласие на обработку персональных данных;
- первая Financial Transaction.

---

### Result

Компания ожидает первую коммерческую оплату.

---

## Phase 6 — First Commercial Payment

### Purpose

Подтвердить начало коммерческих отношений.

---

### Business Events

- получена первая оплата;
- Financial Transaction переходит в состояние Paid.

---

### Result

Возникает основание для создания первого Case.

---

## Phase 7 — Case Creation

### Purpose

Начать выполнение обязательств компании.

---

### Business Events

- создан один или несколько Case;
- для каждого Case автоматически создается Workflow;
- активируется первый Workflow Stage.

---

### Created Artifacts

- Case;
- Workflow;
- первый Workflow Stage.

---

### Result

Request завершает свою основную функцию.

Дальнейшее сопровождение клиента выполняется через соответствующие Case.

# 6. Decision Points

В процессе жизненного цикла Request существуют ключевые точки принятия решений.

---

## Decision Point 1

### Клиент заинтересован в услуге?

**Да**

→ Qualification

**Нет**

→ Request закрывается без дальнейшей обработки.

---

## Decision Point 2

### Клиент принял коммерческое предложение?

**Да**

→ Ready To Start

**Нет**

→ Waiting Decision

---

## Decision Point 3

### Получена первая коммерческая оплата?

**Да**

→ Создать один или несколько Case.

**Нет**

→ Request остается в стадии ожидания.

---

## Decision Point 4

### Требуется создать несколько Case?

**Да**

Создать отдельный Case для каждого Client или Product.

Например:

- семья;
- несколько виз;
- несколько самостоятельных услуг.

**Нет**

Создать один Case.

---

# 7. Completion Rules

Request считается завершенным при выполнении одного из условий.

## Successful Completion

Создан хотя бы один Case.

Request переходит в завершенное состояние и сохраняется как контекст для всех связанных Case.

---

## Closed – Not Interested

Клиент отказался от сотрудничества.

Причина отказа сохраняется в истории Request.

---

## Closed – No Response

В течение 30 дней после отправки коммерческого предложения клиент не вышел на связь.

Request закрывается автоматически.

---

# 8. Automation Rules

Business OS может автоматически выполнять следующие действия.

| Trigger | Automation |
|----------|------------|
| New Request | Создать Request и назначить Manager |
| Qualification Completed | Подготовить коммерческое предложение |
| Waiting Decision (7 дней) | Отправить уважительное напоминание |
| Waiting Decision (30 дней) | Закрыть Request как Closed – No Response |
| Offer Accepted | Подготовить оферту и согласие на обработку персональных данных |
| First Financial Transaction Created | Ожидать поступление оплаты |
| First Commercial Payment | Создать один или несколько Case |
| Case Created | Создать Workflow |
| Workflow Created | Активировать первый Workflow Stage |
